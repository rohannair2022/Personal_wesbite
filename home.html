QuickGather is a social media app designed to simplify event planning with friends. Users can join/create a group by entering their mood, availability, budget, and preferred travel distance. The app then generates group ‘stats,’ providing individual and collective insights to help everyone find common ground and plan events more efficiently.

I had learnt alot building this project. Here are some key learnings and features I gained and implemented :

  • Implemented JWT authentication along with email verification while signing up.

  • Integrated an AWS S3 connection in the Flask REST API backend to store uploaded profile pictures from the frontend, and saved the object link in the user table.
  • Created the entire DB Schema using the SQLALchemy ORM.

Team Project (4) : Open Source Python Package

weedout-package

Python | Pandas | SkLearn | CI/CD | TDD

July - August 2024

Link to PyPi Link to Github

Weedout is an open-source Python package that streamlines data preparation for CSV datasets. It automates encoding, imputation, feature engineering, and data cleaning based on your dataset type and intended model. Weedout's goal is to simplify preprocessing, allowing ML developers to focus on model building.

I built this in a 4 man team and was responsible for the following tasks:

  • Developing and refining multiple functions in the package, leveraging Python libraries such as pandas, scikit-learn, and scipy.
  • Created unit test cases for all functions in the base package, utilizing nose and coverage to implement an effective Test-Driven Development (TDD) strategy.
  • Set up a CI/CD pipeline using GitHub Actions to automate test cases on push and pull requests, and to deploy to the PyPI library after sucessful pull requests

Solo Project : Full Stack Application

weedout-website

Flask | NGINX/ Gunicorn | Unix | EC2 | BDD | HTML/CSS/Bootstrap

August 2024

Link to Site Link to Github

The Weedout website provides a user-friendly GUI for the Weedout package's preprocessing pipeline. Users answer brief questions about their preprocessing needs and upload their dataset. The site then processes the data and returns a zip file with the preprocessed CSV, along with a summary of applied techniques and functions.

These are some key learnings and features I gained and implemented :

  • Migrated and deployed the application to an EC2 instance, utilizing NGINX and Gunicorn for server management
  • Designed an API architecture that processes CSV files, returning a ZIP file containing the preprocessed CSV data.
  • Implemented BDD testing with Behave, using Gherkin syntax to write test cases, and leveraged tools like Selenium and Beautiful Soup for comprehensive testing.
